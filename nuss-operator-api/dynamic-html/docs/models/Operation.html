Operation

  • aircraft_comments : String
    Informative text about the aircraft. Not used by the UTM System.
  • aircraft_type : String
    read-only. Vehicle type propery derived from vehicle registration ID.
  • color_markings : String
  • conformance_time_begin : Date
    The conformance begin time of the operation. Derived from the begin time of the first operation volume.
  • conformance_time_end : Date
    The conformance end time of the operation. Derived from the end time of the last operation volume.
  • created_by : String
    The user that created the operation. It is possible that an operation is created on behalf of an operator by, say, a manager. Nominally, this field will be equal to user_id.
  • decision_time : Date
    A timestamp set by the UTM System any time the state of the operation is updated, for example when the flight goes from PROPOSING to ACCEPTED (see Section 4.1)
  • effective_time_begin : Date
    The effective begin time of the operation. Same value as the begin time of the first operation volume.
  • effective_time_end : Date
    The effective end time of the operation. Same value as the end time of the last operation volume.
  • extra_contact_info : String
    Any additional contact information that may be useful (hours of availability, fax number, communication limitations, etc.).
  • flight_comments : String
    Informative text about the operation. Not used by the UTM System.
  • flight_geography_description : String
    Informative text about the operation. Not used by the UTM System.
  • flight_number : String
    Optional. Currently unused by the UTM System, may be useful to the operator for identification purposes.
  • gufi : String
    has a GUFI assigned upon submission. It is a JSON string that conforms to the UUID version 4 specification (see Section 3.1)
  • operation_volumes : List
    Editable
  • primary_contact_email : String
    Same as primary contact name
  • primary_contact_name : String
    These are required fields but are not currently checked for validity, but clients should endeavor to provide useful, appropriate information in these fields. Validity will be checked in the future. These values should represent the contact that should be used in case of an issue with the operation before, during, or after that operation.
  • primary_contact_phone : String
    Same as primary contact name
  • protected_time_begin : Date
    The protected begin time of the operation. This is derived from the begin time of the first operation volume.
  • protected_time_end : Date
    The protected end time of the operation. This is derived from the end time of the last operation volume.
  • registration_detail : String
    See 'registration' field. This contains details such as the source of the registration ID.
  • registration : String
    The registration ID of the vehicle flying this operation. Note the UTM System assumes a single vehicle per operation currently. This registration value is provided to operators upon manual registration of their vehicle with NASA.
  • secondary_contact_email : String
    Same as secondary contact name
  • secondary_contact_name : String
    These are optional fields. They are not currently checked for validity, but clients should endeavor to provide useful, appropriate information in these fields. Validity will be checked in the future. These values should represent the back-up contact that should be used in case of an issue with the operation before, during, or after that operation.
  • secondary_contact_phone : String
    Same as secondary contact name
  • state : String
    The current state of the operation. Not required for submission, will be assigned by the UTM System.
    Enum:
    P
    A
    R
    C
    X
    I
    E
    G
    B
    U
    T
    F
    V
  • submit_time : Date
    Time the operation submission was received by UTM System.
  • unmanned : Boolean
    Must be set to 'true' with all submissions.
  • user_id : String
    This field is populated based on provided credentials in the HTTPS header.
  • controller_location : Point
  • gcs_location : Point
  • faa_rule : String
    Indication whether this operation is under Part 101-E, Part 107, Part 107 waiver, or a Part TBD. Part TBD is a potential future rule that may cover operations such as those under test by UTM.
    Enum:
    PART_107
    PART_107W
    PART_101E
    PART_TBD
  • waiver_certificate_number : String
    If a waiver has been obtained for the Part 107 rules, then the operator would have a waiver certificate number. For any operation submissions with faa_rule=PART_107W, this field is required.
  • priority_op : Boolean
    if true this operation is managed as a priority operation
