InformMsgType

  • ack_time : String
    The time client acknowledged this message.
  • disclaimer : String
  • free_text : String
    Not used by UTM System. May be formated using Message.Tag
  • gufi : String
    The assigned GUFI for the operation UUID referenced by the message.
  • inform_message : String
    Enum:
    UNKWN
    ACTIVATED
    CLOSED
    CANCELED
    ACCEPTED
    REJECTED
    NONCONFORMING
    EXPIRED
    OP_NOT_SUCCESSFUL
    NOT_ACTIVATED
    ROGUE
    ROGUECLOSED
    ABORTING
    ABORTCLOSED
    CREATE_SUCCESS
    CREATE_FAIL
    UPDATE_SUCCESS
    UPDATE_FAIL
    INVALIDATED
  • message_id : String
    The identifier
  • origin : String
    CLIENT: Message is from a UTM Client to the UTM System.UTM: Message was automatically generated by the UTM System. MANAGER: Message was generated by a UTM Manager (a human).
    Enum:
    CLIENT
    UTM
    MANAGER
  • reason : String
  • sent_time : String
    Either the time the message was sent by the UTM System or the time it was received by the UTM System.
  • surveillance_operation_id : String
  • user : String
  • violations : List
    Included with messages from the INFORM category with inform_message = REJECTED. The array is of pairs of types and constraining_id. The type refers to a constraint in the system (national parks, airports, a volume in an existing operation, wrong registration ID, etc.). The constraining_id is the UUID associated with each constraint; constraining_volume augments it when applicable. When applicable, violating_volume indicates which volume is in violation. This allows for querying UTM for more information about those particular constraints.
  • warning_region_id : String
  • warnings : List
