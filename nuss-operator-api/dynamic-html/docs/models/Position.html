Position

  • air_speed_source : String
    Required if air_speed_track_kn is submitted. No requirements yet on the values here, but suggestions include ESTIMATED or MEASURED.
  • air_speed_track_kn : Double
    Air speed in relation to the direction of travel of the aircraft. Value may be negative.
  • altitude_gps_wgs84_ft : Double
    The altitude as measured via a GPS device on the aircraft. Units in feet using the WGS84 reference system.
  • altitude_num_gps_satellites : Integer
    Number of satellites used in calculating the altitude_gps_wgs84_ft.
  • enroute_positions_id : Integer
  • gufi : String
    Each operation has an GUFI assigned upon submission. Required upon POSTing a new position. It is a JSON string, but conforms to the UUID version 4 specification
  • hdop_gps : Double
    The horizontal dilution of precision as provided by the onboard GPS.
  • location : Point
  • time_measured : Date
    The time the position was measured. Likely the time provided with the GPS position reading.
  • time_received : Date
    Not required for submission, assigned by the UTM System. The time the position was received by the UTM System.
  • time_sent : Date
    The time the position was sent.
  • track_ground_speed_kn : Double
    Ground speed int the direction of travel. Value must be >= 0.0. In knots.
  • track_magnetic_north_deg : Double
    The direction of travel relative to magnetic north in degrees. Value must be >= 0.0 and < 360.0.
  • track_true_north_deg : Double
    The direction of travel relative to true north in degrees. Value must be >= 0.0 and < 360.0.
  • user_id : String
    Not required for submission. This field is populated based on the provided credentials in the HTTPS header.
  • vdop_gps : Double
    The vertical dilultion of precision as provided by the onboard GPS.
